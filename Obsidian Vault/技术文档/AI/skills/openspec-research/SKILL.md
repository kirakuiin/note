# OpenSpec Research — 结构化技术调研

## 定位

Research 是 OpenSpec 流程中 **propose 之后、design 之前（或与 design 并行）** 的环节，用于在动手设计/编码前充分理解现有实现和技术方案：

```
propose → ★ research ★ → design → tasks → apply → verify → review → archive
           搞清楚怎么回事     再决定怎么做
```

- **propose** 确定"要做什么、为什么做"
- **research** 确定"现状是什么、参考实现怎么做的、关键技术点在哪里"
- **design** 基于 research 结论，决定"我们怎么做"

Research 也可以独立使用，不依赖活跃的 change。

---

## 启动方式

### 触发词

- `/opsx:research <topic>` — 指定调研主题
- `调研`、`研究一下`、`帮我调研`、`分析一下实现`、`看看怎么实现的`

### 启动参数

用户需提供以下信息（缺失时主动询问）：

| 参数 | 必选 | 说明 |
|------|------|------|
| **主题/目标** | ✅ | 要研究什么，希望达到什么目的 |
| **参考入口** | 可选 | 已知的入口文件、类名、函数名、关键词等线索 |
| **关联 change** | 可选 | 如果有活跃的 change，research.md 会放入对应 change 目录 |

## 调研流程

### Phase 1: 确定研究目标

1. 与用户确认调研范围和目标
2. 明确核心问题（要回答什么问题）
3. 确定产出预期（为后续哪个环节服务）

### Phase 2: 定位相关文件

1. 根据用户提供的线索（入口文件、关键词等）搜索代码库
2. 使用 grep、语义搜索、文件结构浏览等手段定位相关代码
3. **逐层展开**：从入口点出发，追踪调用链、继承关系、数据流
4. 将发现的文件分类整理（参考实现、目标修改对象、配置/数据、辅助工具等）

### Phase 3: 深入分析实现

对每个关键文件/模块进行分析：

1. **核心数据结构** — 关键的类、结构体、数据容器
2. **核心流程** — 调用链、生命周期、状态流转
3. **关键算法/逻辑** — 核心计算、判断、转换逻辑
4. **接口与协议** — 对外暴露的 API、事件、协议
5. **配置与参数** — 可配置项、常量、开关

分析过程中提取关键代码片段，附上文件路径和行号。

### Phase 4: 归纳结论

1. **技术要点总结** — 核心机制的要点提炼
2. **可复用模式** — 参考实现中可以复用的设计模式、工具函数
3. **风险与注意事项** — 潜在的坑、兼容性问题、性能陷阱
4. **对 design 的建议** — 基于调研结论给出的设计方向建议

## 调研过程中的交互

### 进度展示

每完成一个 Phase，展示当前进度：

```
── Research 进度 ─────────────────────────
  ✅ Phase 1: 确定研究目标
  ✅ Phase 2: 定位相关文件 (发现 N 个相关文件)
  🔄 Phase 3: 深入分析实现 (3/5 模块)
  ⬜ Phase 4: 归纳结论
─────────────────────────────────────────
```

### 中间确认

在以下关键节点与用户确认：

1. **Phase 1 完成后** — 确认研究目标和范围是否准确
2. **Phase 2 完成后** — 展示相关文件列表，询问是否有遗漏或需要调整
3. **Phase 3 过程中** — 发现重要线索时及时同步，询问是否需要深入或调整方向

### 用户可随时

- **调整范围**：扩大或缩小调研范围
- **追加线索**：提供新的入口点或关键词
- **跳过分析**：对某些模块跳过深入分析
- **提前结论**：如果已经获得足够信息，可以提前进入 Phase 4

## 约束

1. **Research 阶段只读** — 不修改任何代码，只阅读和分析
2. **代码片段要注明出处** — 所有引用的代码必须标注文件路径和行号
3. **分析要有深度** — 不能只列出文件名，必须深入到核心逻辑层面
4. **结论要可操作** — 结论和建议必须足够具体，能直接指导后续 design
5. **遵循项目规则** — 分析时参考 .codemaker/rules/ 中的项目约定
6. **控制范围** — 避免无限发散，聚焦于研究目标相关的内容
